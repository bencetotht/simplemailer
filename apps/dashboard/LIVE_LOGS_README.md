# Live Logs Feature

This dashboard now includes real-time WebSocket connectivity to monitor your worker service logs.

## Setup

### 1. Install Dependencies
```bash
cd apps/dashboard
npm install socket.io-client
```

### 2. Start Your Worker Service
```bash
cd apps/worker
npm run start:dev
```
Your worker will run on port 3000 with WebSocket support.

### 3. Start Your Dashboard
```bash
cd apps/dashboard
npm run dev
```
Your dashboard will run on port 3001.

## Features

### Real-time Connection Status
- **Header Indicator**: Shows connection status in the top-right corner
- **Visual Feedback**: Green (connected), Red (error), Gray (disconnected)
- **Quick Actions**: Connect/Disconnect buttons in the header

### Live Logs Page
- **Real-time Updates**: Logs appear instantly as they're generated
- **Auto-subscription**: Automatically subscribes to logs when connected
- **Interactive Controls**: 
  - Ping server to test connection
  - Clear logs
  - Manual connect/disconnect
- **Statistics**: Live counts of total, success, warning, and error logs

### WebSocket Events
The dashboard listens to these events from your worker:
- `connected` - Server welcome message
- `log` - Log entries with level, message, timestamp, and service
- `error` - Connection errors
- `connect`/`disconnect` - Connection state changes

## How It Works

1. **Connection**: Dashboard automatically connects to `ws://localhost:3000` when loaded
2. **Subscription**: Automatically subscribes to log events
3. **Real-time Updates**: Logs appear as they're generated by your worker
4. **Error Handling**: Shows connection errors and allows reconnection

## Configuration

### Worker Service (Port 3000)
- WebSocket gateway runs on the same port as HTTP server
- CORS allows connections from `localhost:3001` (dashboard)
- Supports both WebSocket and polling transports

### Dashboard (Port 3001)
- Connects to worker on port 3000
- Uses Socket.IO client for WebSocket communication
- Handles connection state and errors gracefully

## Troubleshooting

### Connection Issues
1. Ensure worker is running on port 3000
2. Check CORS configuration in worker
3. Verify no firewall blocking localhost connections
4. Check browser console for WebSocket errors

### Missing Logs
1. Verify worker is generating logs
2. Check if logs are being emitted via WebSocket
3. Ensure dashboard is subscribed to log events

### Performance
- Logs are limited to last 100 entries to prevent memory issues
- WebSocket connection is automatically managed
- Reconnection attempts on connection loss

## Development

### Adding New Log Types
1. Update the `LogEntry` interface in `use-websocket.ts`
2. Add corresponding icons and styling in the logs page
3. Handle new log levels in the statistics calculation

### Custom WebSocket Events
1. Add new event listeners in `use-websocket.ts`
2. Create corresponding UI components
3. Update the connection status component if needed
